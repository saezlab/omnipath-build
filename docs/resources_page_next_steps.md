# Resources page: next implementation steps

## Context
- `/resources` in `omnipath-present` now reads from the new `resources.parquet` generated by `omnipath_build`.
- Users can browse, filter, search by resource name, and select one or more resources.
- Selection actions in the UI are still placeholders.

## Step 1: downloads
Implement download support for:
- an individual resource from its card
- the current multi-resource selection from the sticky selection bar

Preferred implementation:
- expose downloads through the FastAPI app
- the FastAPI app should resolve the relevant new build outputs/artifacts for one or more selected resources
- `omnipath-present` should call FastAPI endpoints rather than reading files directly

Need to define:
- what exactly a "resource download" means under the new gold outputs
  - e.g. all gold artifacts for a resource, a zip bundle, selected parquet files, etc.
- response format for multi-resource downloads
  - likely a generated archive from FastAPI

Download packaging guidance:
- do not expect meaningful extra compression from zipping parquet files; parquet is already compressed
- use zip primarily as a bundling/container format, especially when multiple folders or artifacts must be downloaded as one file
- likely approach:
  - single resource -> serve direct artifact if it is just one file, otherwise zip the resource folder/artifact set
  - multi-resource selection -> generate one zip containing the selected resource folders/artifacts

## Step 2: in-app exploration with DuckDB
After downloads are working, allow users to open:
- a single resource
- or a multi-resource selection

for exploration inside the web app using DuckDB.

Likely direction:
- FastAPI provides the selected parquet/artifact set in a form the frontend can consume
- `omnipath-present` reuses or extends the existing DuckDB workspace flow
- the `/resources` page becomes the entry point for selection -> download -> open in DuckDB

## Important
Please align all of this with the new `omnipath_build` outputs and avoid older source-browser / Meilisearch assumptions.
